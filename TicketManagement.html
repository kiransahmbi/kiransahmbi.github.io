Ticket Management Web App

A Web Application that allows users to buy tickets to various events (Concerts, Sports, etc).
The project was completed by designing and implementing an Oracle database.
The database utilizes concepts such as procedures, functions, triggers, and indexes to maintain database integrity upon order creation/deletion.

Requirements

  • User can select a particular event, and show they want to attend. They can select a ticket from a variety of seats and add it to their cart.
  • The User's cart contains all selected tickets and total cost. Any undesired tickets can be deleted.
  • Upon checkout, user will receive an order confirmation. User can view all purchase orders through their Account Page.

Entity Relationship Diagram

Triggers and Procedures/Functions

  1. 'Create Seats' Procedure & Trigger
    • Upon venue creation, the procedure will create all the seat records for the venue.

  2. 'Create Tickets' Procedure & Trigger
    • Upon event-show creation, the procedure will create tickets for all the seats in the venue.
    • The procedure uses the aid of 'Calculate Ticket Price Function'.
      This function calculates ticket prices based on the event type, section number, and aisle number.

  3. 'Create Order' Procedure
    • This procedure is called upon order completion. Corresponding records are created in the ORDERS and ORDER-ITEMS table. It utilizes 'Sales Tax Function' to calculate the regional sales tax.

  4. 'Delete Order' Procedure
    • This procedure is called upon order deletion. Associated records are deleted from the ORDERS and ORDER-ITEMS table.
